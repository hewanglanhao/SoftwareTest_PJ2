## 实现思路

灰盒 fuzzing 的核心挑战之一是：有限的算力应该优先分配给"更有价值"的 seed。本实现采用**路径频率逆调度**策略：

1. 每次运行输入后，记录其**覆盖路径**（coverage 集合的 hash）。
2. 统计每个路径被命中的**频率**。
3. 在 `assign_energy` 时，令 `energy = 1 / freq`——即路径越罕见，该 seed 获得越多变异机会。

### 路径频率记录

`PathPowerSchedule` 维护一个 `path_frequency: Dict[int, int]` 字典：

- key = `hash(frozenset(coverage))`，即当前运行覆盖的代码位置集合的 hash，作为路径的唯一标识。
- value = 该路径被命中的总次数。

`record_path()` 方法在 `PathGreyBoxFuzzer.run()` 中**每次执行**都被调用，确保频率统计覆盖所有运行（包括重复路径），从而真实反映"这条路径有多常见"。

```python
def record_path(self, coverage: Set[Location]) -> None:
    path_id = hash(frozenset(coverage))
    self.path_frequency[path_id] = self.path_frequency.get(path_id, 0) + 1
```

### Energy 分配逻辑

`assign_energy()` 遍历当前种群中的每个 seed，计算其路径并查询频率：

```python
def assign_energy(self, population: Sequence[Seed]) -> None:
    for seed in population:
        path_id = hash(frozenset(seed.coverage))
        freq = self.path_frequency.get(path_id, 1)
        seed.energy = 1.0 / freq if freq > 0 else 1.0
```

- 路径频率越高（常见路径），energy 越低 → 减少探索。
- 路径频率越低（罕见路径），energy 越高 → 增加探索。
- 对于未知路径（`path_frequency` 中不存在），默认 `freq = 1`，给常规 energy。

`PowerSchedule.choose()` 基类会根据归一化后的 energy 做加权随机选择，自动将选择概率偏向高 energy（即罕见路径）的 seed。

### 运行时统计输出

`PathGreyBoxFuzzer` 重写了 `print_stats()`，相比父类增加了两列统计：

| 列 | 含义 |
| --- | --- |
| `Last New Path` | 距离上一次发现新路径的时间 |
| `Total Paths` | 当前已发现的唯一路径总数 |

每次 `run()` 调用时：

1. 记录调用 `super().run()` 前的种群大小。
2. 调用后若种群增大（有新 seed 加入，即发现新覆盖率），更新 `self.last_path_time`。
3. 调用 `self.schedule.record_path()` 记录本次运行的路径。
4. 更新 `self.total_paths = len(self.schedule.path_frequency)`。

这使得运行时可以实时观察路径发现速率——如果 `Last New Path` 长时间不更新，说明 fuzzer 可能陷入了局部探索。

## 与基类的对比

| 方面 | `PowerSchedule`（基类） | `PathPowerSchedule`（本实现） |
| --- | --- | --- |
| energy 分配 | 所有 seed 固定 energy = 1 | energy = 1 / 路径频率 |
| seed 选择偏好 | 均匀随机 | 偏向罕见路径的 seed |
| 路径感知 | 无 | 记录全量路径频率 |
| 调度反馈 | 无 | 每轮运行后更新频率 |

**为什么路径频率调度优于均匀调度？** 均匀调度中所有 seed 获得相同变异次数，已覆盖的常见路径和新发现的罕见路径被无差别对待，导致算力被大量浪费在"已经探索过的区域"上。路径频率调度则让罕见路径的 seed 获得更多变异机会，本质上是将有限的计算资源集中投入在"信息增量更大"的输入上。在覆盖率增长速度和崩溃发现效率两个指标上，路径频率调度通常优于均匀调度，尤其是在 fuzzing 中后期——此时大多数运行都在重复已有路径，均匀调度仍在无差别分配，而路径频率调度已自动将算力倾斜到尚未充分探索的路径。

## 可写入报告的结论

2 号同学实现的路径频率调度已能正常工作。核心机制是 `PathPowerSchedule` 维护全量路径频率字典，`PathGreyBoxFuzzer` 每次运行都记录覆盖路径，使调度器能感知"哪些路径更常见"。`assign_energy()` 将罕见路径的 seed 分配更高 energy，从而在 `choose()` 加权选择时被优先选中，实现"多探索未覆盖区域、少重复已覆盖路径"的调度目标。

运行时统计新增了路径发现维度的可视化，便于在实验报告中观察 fuzzing 过程的路径探索进度。
