# 1号同学 - 变异器实现与测试说明

## 对照要求

本部分按照实验说明和小组分工进行验收，不以本文件自身作为标准。

实验说明中与 1 号同学直接相关的要求是：完善 `simple_fuzzer/utils/mutator.py` 中的变异策略，至少保证多种不同类型的输入变异能够正常工作。

小组分工中 1 号同学负责：检查现有多种变异策略是否稳定工作，补充边界情况测试，必要时增加简单但有效的变异方式，并提供变异策略说明、关键代码、不同样例下的变异效果截图或日志。

因此本模块的目标不是只“写出几个随机函数”，而是提供一个稳定、丰富、可测试、可展示，并且不侵入后续调度和持久化任务的变异器。

## 变异策略

当前 `Mutator` 注册了 15 种变异策略，覆盖字符级、块级、字节级、字典引导、结构引导、数值边界和 havoc 随机扰动。

| 类别 | 函数 | 作用 |
| --- | --- | --- |
| 字符插入 | `insert_random_character` | 在随机位置插入可打印字符，扩大输入长度和字符组合空间。 |
| 字符替换 | `replace_random_character` | 替换一个字符，保持长度不变，适合探索格式边界。 |
| 字符删除 | `delete_random_character` | 删除一个字符，覆盖字段缺失、长度不足等情况。 |
| 块删除 | `delete_random_block` | 删除连续短块，模拟结构损坏和标签缺失。 |
| 块复制 | `duplicate_random_block` | 复制已有片段，制造重复字段、重复标签和长度压力。 |
| 结构插入 | `insert_structure_pair` | 插入括号、引号、花括号、HTML 标签等成对结构。 |
| token 重复 | `repeat_interesting_token` | 重复 `FDU`、`LAB`、`%d`、`{Key}`、HTML 片段等高价值 token。 |
| bit flip | `flip_random_bits` | 翻转 1、2 或 4 个相邻 bit，模拟底层字节扰动。 |
| 字节算术 | `arithmetic_random_bytes` | 对 1、2 或 4 个相邻字节做小范围加减。 |
| interesting values | `interesting_random_bytes` | 将字节替换为 `0x00`、`0xff`、整数上下界等典型边界值。 |
| 字典 token | `overwrite_with_dictionary_token` | 插入或覆盖样例程序相关 token，如 `FDU`、`FDUPLAB`、`<html>`、`{Key}`、`NaN`。 |
| 数字 token | `mutate_numeric_token` | 替换或插入 `0`、`-1`、`2147483647`、`NaN`、`inf` 等数值边界。 |
| havoc 插入 | `havoc_random_insert` | 插入原输入片段或随机可打印字节。 |
| havoc 替换 | `havoc_random_replace` | 用原输入片段或随机字节替换短块。 |
| 块交换 | `random_block_swap` | 交换相邻字节块，改变顺序但保留部分原结构。 |

## 面向样例程序的设计

- `sample1` 依赖 `float(s)` 和数值运算，因此加入数字边界 token：`0`、`1`、`-1`、`NaN`、`inf`、`1e309`、`2147483647` 等。
- `sample2` 依赖 `.` 分割、`{Key}` 和 `%d` 格式化，因此字典和重复 token 中包含 `.`, `%d`, `{Key}`。
- `sample3` 存在 `FDU...L...A...B` 深层分支，因此加入 `FDU`、`FDUPLAB`、`FDUQLAB`、`FDUQLAC`、`LAB` 等 token。
- `sample4` 使用 `HTMLParser`，因此加入 `<html>`、`</html>`、`<script>`、`</script>`、`&amp;` 和成对 HTML 结构。

## 给后续同学铺路

为了方便 2/3/4 号同学做路径调度、策略对比和报告展示，变异器额外提供两个轻量接口：

```python
Mutator().strategy_names()
```

返回当前注册的所有变异策略名，方便统计“某次输入来自哪类变异”。

```python
Mutator().mutate_with_strategy(inp)
```

返回 `(变异结果, 策略名)`，后续如果要做日志、PPT 示例、对比实验，可以直接记录策略来源。

```python
Mutator().generate_examples(inp, count=10, seed=2026)
```

生成固定随机种子的变异示例列表，适合报告或 PPT 中展示“同一个 seed 可产生哪些不同输入”。该方法会恢复外部随机状态，不影响主 fuzz 流程。

这些接口只增强 1 号的变异器可观测性，不改变调度器接口，也不要求后续同学必须使用。

## 边界处理

- 空字符串：删除、替换、bit flip 等不可操作时安全返回；插入、结构、数字、字典类变异可生成非空输入。
- 单字符输入：块操作和字节操作会自动降级到合法范围，避免越界。
- 非 ASCII 输入：字节级变异使用 `utf-8` 转字节，变异后使用 `latin-1` 映射回字符串，避免因解码失败吞掉变异结果。
- 非字符串输入：`Mutator.mutate()` 会先转换为 `str`，避免调用方误传数字或其他对象时直接崩溃。
- 负数示例数：`generate_examples()` 对负数 `count` 主动抛出 `ValueError`，避免产生含义不清的结果。

## 测试覆盖

测试文件 `simple_fuzzer/tests/test_mutator.py` 覆盖以下内容：

- 所有变异函数对空串、单字符、普通字符串、样例格式输入和中文输入均返回字符串且不抛异常。
- `Mutator` 对空串仍能生成可用输入。
- 同一 seed 多次变异能够产生足够多的不同结果。
- 字典变异可确定性注入 `FDU`、`FDUPLAB`、`<html>`、`{Key}`、`NaN` 等关键 token。
- `Mutator` 能处理数字、浮点数、`None`、列表等非字符串输入。
- 注册策略集合与预期 15 种变异策略完全一致。
- `mutate_with_strategy()` 能返回策略名，便于后续统计和展示。
- 数字 token 变异能替换已有数字，也能在没有数字时插入边界值。
- 结构插入能生成 HTML 成对标签。
- token 重复能放大 `FDU` 等关键分支 token。
- `generate_examples()` 在固定 seed 下可复现。
- `generate_examples()` 对负数 `count` 会抛出异常。
- `generate_examples()` 使用固定 seed 时会恢复外部随机状态，不污染其他模块。

## 可写入报告的结论

1 号同学负责的变异器已经超出“多种变异方式可正常工作”的基础要求。当前实现不仅包含常见随机插入、删除、bit flip、算术变异和 havoc，还针对实验提供的 4 个 sample 设计了字典引导、数值边界、结构插入和 token 重复等更有目标性的策略。

测试方面，当前不是只验证“能跑”，而是验证了边界输入、非字符串输入、策略注册完整性、关键 token 注入、示例可复现性和随机状态隔离。这样既能支撑 1 号同学自己的展示，也能为后续路径调度、结果对比和实验报告提供更稳定的输入生成基础。
