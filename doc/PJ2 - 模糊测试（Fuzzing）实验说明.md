# PJ2 - 模糊测试（Fuzzing）实验说明

## 一、实验目标

本实验的目标是让同学们理解一个简化版灰盒模糊测试器的工作方式，并在现有框架上补齐关键能力。你们会接触到种子输入、输入变异、覆盖率反馈、路径调度和结果持久化等内容。

如果你想先快速了解项目，请先看 [simple_fuzzer/README.md](simple_fuzzer/README.md)。README 负责“说明书”，本文负责“作业单”。

## 二、项目大致内容

本项目提供了一个可运行的 fuzzing 框架骨架，主要包含以下部分：

- `fuzzer`：负责输入生成与 fuzz 主流程。
- `runner`：负责执行目标函数并采集覆盖率。
- `schedule`：负责决定“哪个 seed 更值得继续变异”。
- `utils`：提供覆盖率追踪、变异器、种子对象和对象持久化工具。
- `samples`：提供 4 个示例被测程序，供同学们验证 fuzz 效果。

项目的设计引导请以 [simple_fuzzer/README.md](simple_fuzzer/README.md) 为准。本文不重复展开底层设计细节，避免你在实验前就被代码树“劝退”。

## 三、编码任务

请在现有框架基础上完成以下内容：

1. 完善 `mutator.py` 中的变异策略，至少保证多种不同类型的输入变异能够正常工作。
2. 完善 `path_power_schedule.py` 与 `path_grey_box_fuzzer.py`，实现基于路径频率的 seed 调度逻辑。
3. 增加至少一种新的调度策略，并放入 `schedule` 包中。
4. 使用 `object_utils` 将 seed 或相关中间结果持久化到文件系统，避免长时间运行时内存持续增长。

> 如果你能资助助教一条内存，那么我可以认为你已经完成了持久化的任务了（误）。
